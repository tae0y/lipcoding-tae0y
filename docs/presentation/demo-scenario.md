## 0. 한 줄 메시지 (오프닝 멘트, 15초)

> "사람의 작업기억은 한 번에 5±2개. 떠오른 아이디어를 붙잡고 있으면 그 한계를
> 갉아먹습니다. 이 앱은 아이디어를 **던지면**, AI가 *지금 착수 가능한지*를
> 판정하고, 정보가 부족하면 **대신 사전조사**까지 해둡니다. 계획·판단·실행은
> 사람이 유지하고, AI는 바닥만 깝니다 — **대체가 아니라 강화**."

## 1. 데모 전 준비 (무대 뒤 체크리스트)

| 항목 | 확인 |
|------|------|
| Azure URL 접속 → 앱 첫 화면 로딩 | `GET /` |
| AI 모델 계층 생존 확인 | `GET /health/ai` 가 `{ ok: true }` |
| 시드 데이터 준비 (덤프 ≥ 2건, 인박스 ≥ 1건) | 조회 화면에서 미리 확인 |
| 오늘 상태 초기값: 저녁 비었음 ✓ / 기분 보통 | 게이트 통과 조건 맞춰둠 |
| 네트워크 끊김 대비: 로컬 백엔드 폴백 1개 띄워둠 | — |

> 시드를 미리 넣어 두면 "이번 주 추천"과 덤프 사전조사 노트를 즉시 보여줄 수
> 있어 라이브 리스크가 준다.

## 2. 메인 시나리오 (약 3분, 한 흐름)

### 장면 1 — 착수 가능한 아이디어 캡처 (입구 판정 ✅)

1. **입력 화면**에서 아이디어 한 줄 입력:
   > `깃허브 프로필 README에 이번 프로젝트 한 줄 추가하기`
2. **[ 담기 ]** 클릭.
3. → AI 입구 판정(Copilot SDK)이 **정보 준비성 + 현재 여유** 2요소를 평가.
4. 결과 피드백: **"지금 착수 가능 → 인박스에 추가됨"**.

> **멘트:** "다음 액션이 분명하고 지금 여유도 되니, AI가 바로 인박스로
> 올렸습니다. AI가 한 건 *판정*이지 대신 *해주는 게* 아닙니다."

### 장면 2 — 정보가 부족한 아이디어 → 덤프 + 자동 사전조사 (SDK 깊이 🔧)

1. 이어서 모호한 아이디어 입력:
   > `사이드 프로젝트로 RAG 챗봇 만들어보기`
2. **[ 담기 ]** 클릭.
3. → 판정 결과: **"덤프 보관 [info_gap]"** + 그 자리에서
   **"AI가 사전조사 중…"** 스트리밍 시작.
4. 스트리밍이 흐르며 `materials`(참고 사실·링크)와 `options`(A/B/C 선택지)가
   채워진다.

> **멘트:** "정보가 부족하면 보류하되, AI가 **재료와 선택지**까지 정리해 둡니다.
> 단, **'다음 액션 확정'은 일부러 안 만듭니다** — 그건 사람 몫입니다.
> 이게 augment의 경계선입니다." *(Responsible AI 6% 어필)*

### 장면 3 — 조회 화면: 한 스크롤로 우선순위 보기 (UX 12%)

1. 상단 **[ 조회 ▸ ]**로 이동.
2. 위에서 아래로 한 번 스크롤하며 세 섹션을 보여준다:
   - **이번 주 추천** — 카드 + 근거 3줄 + 사람 결정 버튼.
   - **지금 착수 가능 (inbox)** — 방금 담은 README 항목 포함.
   - **보관함 / 덤프** — RAG 항목의 `[info_gap]` 배지 + **▸조사노트 펼치기**.
3. 덤프의 **▸조사노트**를 펼쳐 방금 AI가 채운 `materials`/`options`를 보여준다.

> **멘트:** "탭 없이 한 스크롤. 위에서 아래로 갈수록 '지금 할 것 → 나중'.
> 회색 배지로 *왜 보류됐는지* 투명하게 보입니다."

### 장면 4 — 주간 추천 트리거 + 사람의 최종 승인 (Responsible AI 🙋)

1. **이번 주 추천** 카드를 가리키며 근거 3줄을 읽는다:
   - `lowLoad` — 저녁이 비었고 기분이 받쳐줌
   - `researchDone` — 사전조사 재료·옵션 정리됨
   - `relevance` — 최근 관심사와 연관 (덤프 ≥ 5건일 때 임베딩 유사도)
2. **[ 지금 해볼게요 ]** 클릭 → `decision: accepted`.

> **멘트:** "추천은 AI가 하지만 **결정 버튼은 항상 사람**입니다. 부하 게이트
> (`저녁 비었음 AND 기분≠나쁨`)를 통과할 때만 한 주에 **딱 한 건** 제안해
> 작업기억을 더 어지럽히지 않습니다."

### 장면 5 — Azure 위에서 돈다 (Azure AI 18%, 클로징)

1. (선택) 새 탭에서 `GET /health/ai` 응답 `{ ok: true, reply: ... }`를 보여준다.

> **클로징 멘트:** "판정·사전조사·연관성 매칭 전부 **Azure OpenAI** 위에서
> 돕니다. 데이터는 Azure Files 볼륨의 SQLite에 살아남고, 앱은 Azure Container
> Apps에 배포돼 있습니다. — 작업기억은 사람이, 바닥 까는 일은 AI가."

## 3. 심사 기준 ↔ 데모 장면 매핑 (멘트 근거)

| 심사 항목 | 가중치 | 데모에서 보여주는 지점 |
|------|---:|------|
| Effective Use of Copilot SDK | 25% | 장면 1 입구 판정 도구 · 장면 2 사전조사 스트리밍(도구 체인) |
| Productivity & Problem Fit | 18% | 오프닝 작업기억 프레이밍 · 장면 4 "주 1회 1건" |
| Azure AI & Cloud Integration | 18% | 장면 5 `/health/ai` · 임베딩 연관성 랭킹 |
| Functionality & Technical Execution | 16% | 장면 1→4 끊김 없는 엔드투엔드 |
| UX & Workflow Design | 12% | 장면 3 한 스크롤 · 스트리밍/로딩 처리 |
| Responsible AI, Security & Trust | 6% | 장면 2 "다음 액션은 사람" · 장면 4 사람 승인 버튼 · secret 환경변수 |
| Innovation & Originality | 5% | 작업기억 5±2 인지과학 프레이밍 |

## 4. 백업 플랜 (라이브 사고 대응)

| 상황 | 대응 |
|------|------|
| Azure URL 무응답 | 로컬 백엔드 폴백으로 즉시 전환(동일 화면) |
| AI 스트리밍 지연/실패 | 미리 시드해 둔 덤프의 `▸조사노트`로 결과만 보여주고 멘트 지속 |
| 입구 판정이 의도와 다른 분류 | "AI 판단이라 경계 사례가 있다"고 인정 후 다른 시드 항목으로 전환 |
| 추천 카드가 비어 있음 | 입력 화면에서 저녁 토글 ✓ + 기분 보통 확인 → `POST /suggestions/run` 재실행 |
| 네트워크 전면 다운 | 스크린샷/녹화 클립으로 흐름 설명 (사전 캡처본 준비) |
