## 앱 심사 기준

| # | 항목                                | 가중치   |
|---|-------------------------------------|---------:|
| 1 | Effective Use of Copilot SDK        |      25% |
| 2 | Productivity Impact & Problem Fit   |      18% |
| 3 | Azure AI & Cloud Integration        |      18% |
| 4 | Functionality & Technical Execution |      16% |
| 5 | User Experience & Workflow Design   |      12% |
| 6 | Responsible AI, Security & Trust    |       6% |
| 7 | Innovation & Originality            |       5% |
|   | **Total**                           | **100%** |

---

## 1. Effective Use of Copilot SDK — Weight: 25%

Copilot SDK가 앱의 가치에 핵심적인 역할을 합니까? 프롬프트/에이전트 설계, 도구 호출, 컨텍스트 처리, 스트리밍의 품질과 적절성 — 기능의 개수보다 깊이를 평가합니다.

## 2. Productivity Impact & Problem Fit — Weight: 18%

명확한 대상 사용자에게 실질적이고 입증된 이점을 제공하면서, 현실적이고 잘 정의된 생산성 문제를 해결합니까?

## 3. Azure AI & Cloud Integration — Weight: 18%

실제로 얼마나 의미있게 Azure 클라우드를 활용하는지를 평가합니다. 최고 점수를 받을 수 있는 비결은 AI/모델 계층이 Microsoft Foundry / Azure OpenAI 위에서 동작해야 합니다. 클라우드 네이티브 프랙티스 관련성에 가점을 주고, 단순 Azure 서비스 끼워넣기는 감점합니다.

## 4. Functionality & Technical Execution — Weight: 16%

애플리케이션은 엔드투엔드로 작동합니까? 코드 품질, 아키텍처, 오류 처리, 성능, 그리고 플랫폼에 적합한 구현(네이티브 크로스플랫폼 또는 반응형 웹) 등을 평가합니다.

## 5. User Experience & Workflow Design — Weight: 12%

사용자의 저항이 낮은 UI, 자연스러운 AI 통합, 지연/오류/투명성에 대한 우아한 처리, 접근성, 그리고 사용자가 주도권을 유지하도록 하는 설계를 평가합니다.

## 6. Responsible AI, Security & Trust — Weight: 6%

AI 출력의 투명성, 위험한 작업 전 사람의 확인, 데이터 프라이버시, 환각(hallucination) 완화, 프롬프트 인젝션에 대한 인지, 그리고 안전한 비밀 정보(secret) 처리를 평가합니다.

## 7. Innovation & Originality — Weight: 5%

기존 서비스/도구 등의 뻔한 복제보다 AI를 어떻게 생산성 측면에서 새롭고 창의적으로 적용했는지를 평가합니다.
