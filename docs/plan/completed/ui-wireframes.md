# 화면 와이어프레임 (ASCII)

> 그릴링 세션(2026-06-20)에서 확정한 화면 레이아웃. 구현 전 참조용이며, 변경 시
> 이 문서를 먼저 갱신한다. 상위 요구사항은 [app-requirements.md](./app-requirements.md),
> API는 [openapi.yaml](./openapi.yaml), 비주얼 스타일은
> [../research/design.md](../research/design.md) 참조.

**비주얼 = 빅테크 미니멀(shadcn `neutral` 디폴트).** 이모지 장식·컬러 배지 없음.
상태/사유는 **회색 배지**(`secondary`), 파괴 액션만 `destructive`. 강조색은 `primary` 1개.
아래 아스키의 아이콘/이모지는 위치 표시일 뿐, 실제 구현은 텍스트+회색 배지로 한다.

화면은 **2개**(입력 / 조회)로 고정. 탭 전환 없이 각 화면은 단일 스크롤,
본문 폭 ~640px 컨테이너, `Card`/`Button`/`Textarea`/`Badge` 프리미티브만 사용.

> design.md의 "3개 논리 영역"(인박스 · 덤프 · 제안) → 본 문서의 **2화면**에 매핑:
> 입력 화면 = 캡처/판정, 조회 화면 = 제안 + 인박스 + 덤프(한 스크롤).

---

## 화면 1 — 입력 (캡처 + 오늘 상태)

```text
┌────────────────────────────────────────────────────────┐
│  Ideas                                       [ 조회 ▸ ] │   ← 얇은 상단바
├────────────────────────────────────────────────────────┤
│                                                        │
│  아이디어 한 줄                                         │   ← 주 기능
│  ┌──────────────────────────────────────────────────┐ │
│  │  여기에 떠오른 생각을 한 줄로...                    │ │   ← Textarea
│  └──────────────────────────────────────────────────┘ │
│                                   [ 담기 ]              │   ← primary 버튼
│                                                        │
│  ── 판정 결과 ──────────────────────────────────────── │
│  ┌──────────────────────────────────────────────────┐ │
│  │ 지금 착수 가능 → 인박스에 추가됨                    │ │
│  │   또는                                            │ │
│  │ 덤프 보관  [info_gap]    AI가 사전조사 중…▌        │ │   ← 스트리밍
│  │ 덤프 보관  [no_capacity] 여유 생기면 추천할게요     │ │
│  └──────────────────────────────────────────────────┘ │
│                                                        │
│  오늘 상태                                              │   ← UserState
│  ┌──────────────────────────────────────────────────┐ │
│  │  기분    [ 나쁨 ] [ 보통•] [ 좋음 ]                │ │   ← 3버튼(토글)
│  │  저녁    [✓] 저녁 비었음  ( 끄면 = 약속 있음 )     │ │   ← calendarBusy
│  │  할일    ┌────────────────────────┐ [+ 추가]      │ │   ← todos
│  │          │ • 장보기                │              │ │
│  │          │ • 운동                  │              │ │
│  │          └────────────────────────┘              │ │
│  └──────────────────────────────────────────────────┘ │
│                                                        │
│  주간 추천 시각                                         │   ← triggerSchedule
│  ┌──────────────────────────────────────────────────┐ │
│  │  요일 [ 금 ▾ ]   시각 [ 19:30 ]                    │ │
│  │  (퇴근 + 20~30분 버퍼 권장)                        │ │
│  └──────────────────────────────────────────────────┘ │
│                                                        │
└────────────────────────────────────────────────────────┘
```

입력 항목 매핑:

| UI | 데이터 | 비고 |
|----|--------|------|
| 아이디어 한 줄 | `Idea.text` | 담기 → 입구 판정 즉시 실행 |
| 기분 3버튼 | `UserState.emotion` (bad/normal/good) | 마지막 값 유지 |
| 저녁 토글 | `UserState.calendarBusy` (boolean) | 외부 캘린더 연동 없음 |
| 할일 입력 | `UserState.todos` (string[]) | 앱 내 직접 입력 |
| 요일·시각 | `UserState.triggerSchedule` | 주 1회 배치 트리거 |

## 화면 2 — 조회 (한 스크롤: 추천 → 인박스 → 덤프)

```text
┌────────────────────────────────────────────────────────┐
│  Inbox                                       [ + 입력 ] │   ← 얇은 상단바
├────────────────────────────────────────────────────────┤
│                                                        │
│  이번 주 추천                                           │   ← 섹션 1 (트리거 결과)
│  ┌──────────────────────────────────────────────────┐ │
│  │  "주말에 텃밭용 화분 만들기"                        │ │
│  │  ───────────────────────────────────────────────  │ │
│  │  부하 낮음   저녁 비었고 기분 보통         (lowLoad)│ │
│  │  사전조사 완료 재료·옵션 정리됨 ▸펼치기 (researchDone)│ │
│  │  관심사 연관  최근 '베란다 가드닝'과 유사 (relevance)│ │
│  │  ───────────────────────────────────────────────  │ │
│  │   [ 지금 해볼게요 ]  [ 다음에 ]  [ 관심없음 ]       │ │
│  └──────────────────────────────────────────────────┘ │
│                                                        │
│  지금 착수 가능 (inbox)                            3건  │   ← 섹션 2
│  ┌──────────────────────────────────────────────────┐ │
│  │ • 블로그 글 "작업기억 5개" 초안 쓰기               │ │
│  │ • 깃허브 프로필 README 갱신                        │ │
│  │ • 동네 러닝 코스 한 바퀴 정하기                    │ │
│  └──────────────────────────────────────────────────┘ │
│                                                        │
│  보관함 / 덤프                                     5건  │   ← 섹션 3
│  ┌──────────────────────────────────────────────────┐ │
│  │ • 사이드 프로젝트 아이디어   [info_gap]   ▸조사노트 │ │
│  │     └▸ 재료(materials): 참고링크 2 · 옵션: A/B/C    │ │
│  │ • 새 키보드 알아보기         [no_capacity]         │ │
│  │ • 전시회 가보기              [no_capacity]         │ │
│  │ • RAG 논문 정리             [info_gap]   ▸조사노트 │ │
│  │ • 캠핑 장비 점검             [no_capacity]         │ │
│  └──────────────────────────────────────────────────┘ │
│                                                        │
└────────────────────────────────────────────────────────┘
        ▲ 위에서 아래로 한 번에 스크롤 (탭 전환 없음)
```

레이아웃 의도:

- **위 → 아래 = 행동 우선순위순** (추천 → 바로 할 것 → 보관).
- 추천 카드만 강조 `Card` + 근거 3줄(`reasons.lowLoad`/`researchDone`/`relevance`)
  + 사람 결정 버튼. 추가 색·강조선 없이 여백·위계로만 구분.
- 덤프의 `info_gap` 항목만 `▸조사노트` 펼치기 → `research.materials`(fact/url) + `research.options`.
  `no_capacity`는 회색 배지만.
- 사유 태그(`info_gap`/`no_capacity`)는 우측 **회색 배지**(`secondary`) → 투명성(심사 #5).
- 결정 버튼은 항상 사람 (`accepted` / `postponed` / `dismissed`) — 강화 원칙.

## 확정된 관련 결정

- 비주얼: **빅테크 미니멀(shadcn `neutral` 디폴트)** — 이모지·커스텀 컬러 없음, 회색 배지 + `primary` 1색.
- 레이아웃: **섹션 한 스크롤** (탭 분리 아님).
- 추천 건수: **1건만**.
- 외부 연동 없음: 캘린더·투두·감정 전부 앱 내 수동 입력.
